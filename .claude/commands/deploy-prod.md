⚠️ 生产环境部署 — 请确认已完成灰度验证！

将当前特性分支通过 PR 合并到 prod 分支，触发生产环境部署。

## 前置检查

1. 确认代码已在灰度环境（release 分支）验证通过
2. 确认当前不在 prod/test 分支上

## 执行步骤

1. `git push origin {当前分支名}` 确保远程有最新代码
2. 创建 PR: `gh pr create --base prod --head {当前分支名} --title "类型: 描述" --body "## 变更说明\n\n- \n\n## 测试情况\n\n- 测试环境已验证\n- 灰度环境已验证"`
3. 提醒用户去 GitHub 确认 PR 并合并

## 注意

- **不要直接 push 到 prod！** 必须通过 PR 合并
- PR 会自动触发 AI 代码评审
- 合并后流水线自动部署到生产环境
